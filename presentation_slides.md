# Слайд 1: Проблема и Цель
**MCP-сервер "DevBoost"**
* **Боль:** "Залипшие" порты ломают локальный запуск, а отсутствие индексов в БД (Seq Scan) вешает весь бэкенд на проде.
* **Почему эта боль:** Разработчики ежедневно тратят до 2 часов на дебаг окружения и чтение логов `EXPLAIN ANALYZE`. Наш MCP делегирует эту рутину AI-агенту.
* **Цель:** Создать сервер, превращающий любого ИИ (Claude/Cursor) в локального DevOps и DBA.

---

# Слайд 2: Схема агентной системы
* **Роли:**
  * **Человек (Product Owner / Reviewer):** Ставит бизнес-цели, проверяет код, принимает архитектурные решения.
  * **AI-Агент (DeepMind Antigravity / Claude 3.5 Sonnet):** Выступает как ведущий разработчик (Tech Lead) и кодер (Execution).
* **Инструменты:**
  * IDE, Терминал, Docker.
  * Artifacts: `task.md` (чек-лист), `implementation_plan.md` (спецификация контрактов).

---

# Слайд 3: Процесс работы (Осознанный пайплайн)
Работа шла не хаотичным промптингом, а по строгому циклу:
1. **Planning (Анализ & Проектирование):** Формирование проблемы, создание `implementation_plan.md` и согласование API инструментов (интерфейсов MCP).
2. **Execution (Разработка):** Написание Python-кода, реализация 4 инструментов (sys_process, db_client).
3. **Verification (QA & Ревью):** Поднятие Docker-контейнера с тестовой БД на 100k записей. Ручное тестирование Inspector'ом.
4. **Correction:** Возврат в Execution при ошибках с `sse` транспортом.

# Слайд 4: Контроль качества и Валидация
* **Изолированное тестирование:** Был создан `demo_project` с БД и скриптом генерации 100 000 строк без индексов (искусственное замедление).
* **Многоуровневая проверка:**
  * Smoke-тесты при сборке Docker (`/health`).
  * Unit-тесты не писались ради скорости хакатона, но инструменты тестировались "в бою" через MCP Inspector.
  * Ручное ревью (Человек) подтверждало, что AI не удаляет файлы (защита безопасности в `kill_port_hog`).

# Слайд 5: Использованные инструменты при разработке
* **MCP SDK (Python):** Для быстрого старта (библиотека `fastmcp`).
* **Starlette / Uvicorn:** Для организации SSE транспорта и кастомных роутов `/health`.
* **lsof / bash:** Под капотом для работы с процессами ОС.
* **Docker / Compose:** Универсальная упаковка демо-стенда, независимая от ОС жюри.
